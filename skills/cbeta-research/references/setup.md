# CBETA MCP Setup

## Submodule

The server lives at:

```bash
mcp_servers/CbetaMCP
```

Initialize after cloning this repository:

```bash
git submodule update --init --recursive
```

## Python Environment

Install dependencies explicitly when needed:

```bash
cd mcp_servers/CbetaMCP
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

Do not commit `.venv`, caches, logs, or generated bytecode.

## Endpoint

Use port `18765` by default:

```bash
APP_PORT=18765 python main.py
```

Default MCP URL:

```text
http://localhost:18765/mcp/
```

Example MCP config:

```json
{
  "mcpServers": {
    "cbeta": {
      "url": "http://localhost:18765/mcp/",
      "type": "streamable_http",
      "description": "CBETA Buddhist Scripture Search MCP Server"
    }
  }
}
```

## Diagnostics

Check server availability:

```bash
python3 skills/cbeta-research/scripts/check_server.py --url http://localhost:18765/mcp/
```

Start on demand:

```bash
skills/cbeta-research/scripts/start_server.sh
```

Use an explicit Python environment if dependencies are already installed elsewhere:

```bash
CBETA_MCP_PYTHON=/path/to/venv/bin/python skills/cbeta-research/scripts/start_server.sh
```

Use foreground mode when the execution environment cleans up background processes:

```bash
CBETA_MCP_FOREGROUND=1 skills/cbeta-research/scripts/start_server.sh
```

The startup script writes logs to `/tmp/cbeta-mcp.log` by default.

## Windows and WSL2

Prefer WSL2 for CBETA MCP on Windows because the startup script and setup commands assume a Unix-like shell.

If the agent is already running in WSL2:

```bash
cd /path/to/everything-analysis-agent
skills/cbeta-research/scripts/start_server.sh
```

If the agent is running in native Windows PowerShell or CMD and WSL2 is available:

```powershell
wsl.exe -- bash -lc 'cd /mnt/c/path/to/everything-analysis-agent && skills/cbeta-research/scripts/start_server.sh'
```

If the repository lives inside the WSL filesystem, use its Linux path instead of `/mnt/c/...`.

Use native Windows only when WSL2 is unavailable. In that case, start the server with PowerShell:

```powershell
.\skills\cbeta-research\scripts\start_server.ps1
```

Use foreground mode when the execution environment cleans up background processes:

```powershell
.\skills\cbeta-research\scripts\start_server.ps1 -Foreground
```

The script creates `.venv` only when you do it explicitly. If dependencies are missing, run:

```powershell
cd mcp_servers\CbetaMCP
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```
